    Generate bitmap by Text

    Bitmap is the image combined by individual pixel. To reproduce a bitmap, we can analyze each value of pixel. Which means we should get the color of this pixel, when we generate new pixel / graphic, we can use these data, so the new image will looks like the original one.

    位图是由像素组成的,为了重绘一个位图,我们要分析每一个像素的数据比如颜色和位置. 当得到那些数据后,在绘制的时候,就能把相对应的数据带入新的像素(这可能是由字符或者别的图形组成的), 这样的新图片看起来就会和原来的图片很像.

    Environment

    In this workshop, I’m going to use HTML and Javascript to build the drawing programme. To control Canvas in HTML5, I used CreateJS to draw and set graphic, which eaiser than original Web canvas API.

    Get pixel data

    Firstly, we should create image then add it in context of canvas.

    let ctx = document.getElementByID('canvas ID').getContext('2d')
let img = new Image()
img.src = 'img/xxx.jpg'

ctx.drawImg(img,0,0)

    So far, we’ve draw the image on canvas, so we can call getImageData to build a object which save data of pixel in memory.

    let imgDt = ctx.getImageData(0,0,img.width,img.height)
// chose a range of area, same as image's size

    draw pixel one by one

    Before we use these data to draw, we should relize what’s it. getImageData() built a ImageData object, this object has a data attribute, and all of pixel data have saved in there as a Uint8ClampedArray, which only save the number in [0-255]. And our pixel will be analyze the color as RGBA. So each pixel will add a number in this Array.

    These a article I read, it analyzed the regular of ArrayLink

    For draw pixel at X and Y axis, we should use two For() loop to create graphic in 2D.

    for(let v=0;v<img.width;v++){
    for(let row=0;row<img.height;row++){
        // these two for loop can iterate each pixel
        // key point is to get index from IMGdata

        let index = [v*img.width + row]*4
        imgDt[index] //color R
        imgDt[index+1 ] //color G
        imgDt[index+2 ] //color B
        imgDt[index+3 ] //color A, influence size

        // there're all of pixel data

    Blog development

     workshop

    Author:Hayden

    2019-02-20

    Blogging!

    This blog began from Oct 2018, which is the time I started my MA course. At early stage, I used blog to record some experiment and notes on seminar. It is a good way to save and share ideas though nobody ever came here😢😢, it is still a good place I can record my academic life. furthermore, I develop my Blog by Hexo framework, so I learn much knowledge about Web development. And I’m developing more function and style by myself, which real good for my coding ability and career. Let’s talk about more about my Blog construction, function, and future expectation.

    Categories

    Category is important to sort post. I used different Caties(categories) to define different function of articles / posts. different caties have individual style and layout, and they have differnt content on index, which support clear information to audience or myself.

    Journal

    This is the essential of blog, most post were defined as Journal, which carry general content, such as note, ideas, experiment, anything else. It could not be academic, and sometime will be post images. So, the layout on index can be divided as two style, preview content& image and abstract.

    gallery

    This is a important class for a visual artist. It shows preview image on index for direct visual expression. Furthermore, the image can be clicked as new block, user can click button to change and scan image of a post. It basically is used to post my outcome or visual experiment.

    As the increasement of image, I found I might need more style to show differnt kinds of image, such as drawing, photography, vector graphic, motion graphic, video. They are definitely differnt, when I exhibit them, should I use differnt size / resolution ? Maybe. So I add some options in module programme, when code checked differnt type, it generate relateive layout.

    research

    As digital artist, I would like do research on computer, as well as experiment, so most recording and research article are saved on computer. Then I create this cat as my sketchbook, all of file or text / image can saved and post on research cat, so peers and tutor can easily relize what Im doing. Because it could be simple or short, sometimes massive text from copy and reference, so I decided to use Tag to abstract content of post as research cat. Which means, index only show tag of this kind of post, I and others can click tag to gain more relatvie information from same tag of post. It is a good way to expand idea horizontally.

    reflection

    Reflection will be academic when I post that. In future, I will add Grammaly or Translate API to help different language user to understand my article. On index, reflection block will show preview content and catelog and content guide, which means, audience can click a interesting Part / heading of content, and he will go to the sepcific postion of this post. By this way, audience could quickly understand what I’ve done, what steps I used, thinking process.

    Other expectation

    I’m going to develop a new cat which can collect ideas. Sometimes I used journal to record idea, but idea could be really short and shuffle, it can not be a article, and it will be burden(累赘) for index, if too many short post are displayed on index. That’s the reason I want to build a idea Page, in this page, all of idea will be grouped and showed, with relative tag, is it sounds really cool? I can shuffle these ideas and randomly scan them when I want to find inspiration. Maybe I can add some function to allow user select some words, and programme will use google search, and find some interdisciplinary research.
